Recast.mt
Manual — v2 / 2026
01
Overview

Recast.mt is a tool for using LLMs as pure transformation functions rather than conversational agents.

Instead of instructing an AI in natural language each time, you select a transform mode and configure parameters. Save them as a preset and re-run the same transformation in one tap.

Supports three providers: Anthropic, OpenAI, and Gemini. Any provider works as long as you have a valid API key.

02
Requirements
API key
At least one API key from OpenAI, Anthropic, or Gemini is required. Keys are stored only in the browser's localStorage — never sent to any external server.
Browser
Latest Chrome, Safari, or Firefox recommended. Works on mobile.
PWA install
Add to home screen to launch as a standalone app. Optional.
Billing
Your API key must be active and API access enabled with the provider. Free tier works within usage limits.
03
Getting Started
01
Set your API key

Tap ⚙ in the top right, enter your API key in the API KEYS section, then press save keys.

02
Create a preset

In the same settings screen, scroll to PRESET. Set a name, provider, model, transform mode, output language, and parameters. Press save new.

03
Run a conversion

Close settings. Select your preset from the dropdown, paste text into INPUT, then press CONVERT.

04
Preset Operations
load preset
Select an existing preset from the dropdown at the top of the settings screen. The form is populated with its values immediately.
— new preset —
Resets the form to default values. Use this to start building a new preset from scratch.
save new
Saves the current form as a new preset.
overwrite
Overwrites the currently selected preset with the current form values. Useful for tweaking parameters and saving the updated version.
delete
Deletes the currently selected preset.
05
Transform Modes
— structural
summarize
Extract key information and shorten
expand
Elaborate each element in detail
outline
Convert to hierarchical outline structure
bulletize
Convert to bullet points (no evaluative language)
compress
Preserve meaning with minimum words
formalize
Convert to formal or official register
simplify
Convert to plain vocabulary and structure
abstract
Remove specific details and abstract
— semantic
extract_claims
List explicit and implicit claims
extract_assumptions
List underlying assumptions
extract_structure
Describe logical structure and relationships
remove_evaluation
Strip evaluative and emotional language, keep factual statements only
neutralize
Remove stance and value judgments, convert to neutral description
invert
Reverse the argument or position
— format
json
Convert to JSON (JSON enforced at API level)
yaml
Convert to YAML
table
Convert to Markdown table
markdown
Convert to Markdown format
pseudo_code
Convert to pseudocode
— language
translate
Translate to the language specified in output language
06
Parameters
provider
The LLM provider to use: anthropic, openai, or gemini. The corresponding API key must be set.
model
The model to use. Options update when the provider is changed. For structural transformations, lighter models tend to follow instructions more literally.
output language
日本語 or English. In translate mode, this setting determines the target language.
temperature
Output randomness. Range: 0.0–1.0. Lower values (0.1–0.3) work well for structural and JSON conversions. Higher values (0.7–1.0) suit creative transformations. Default: 0.7.
max_tokens
Maximum output token count. Increase for longer input texts. Default: 1000.
07
Known Limitations
Anthropic
Direct browser requests to the Anthropic API are blocked by CORS. A Cloudflare Workers proxy must be configured separately for Anthropic to work.
Gemini 2.5-pro
May return empty responses in JSON mode. Try increasing max_tokens or switching to a different mode and retrying.
API key storage
Keys are stored as plain text in localStorage. Not recommended on shared devices.
Conversation history
Each conversion is a stateless, single-shot API call. The output of a previous conversion cannot be referenced in the next one.
Design intent

This app does not treat LLMs as conversational partners. It treats them as transformation functions applied to input text. Persona, empathy, and continuation prompts are intentionally absent.
